# ai-gm-core 日本語まとめ（レイナ＆シュン用）

## これは何か

ai-gm-core は「ゲーム」を作るためのプロジェクトではない。

**AIを“ゲームマスター（GM）”として扱うための、思考の芯（コア）を置く場所**である。

目的は、

* 勝ち負けを決めること
* 正解を提示すること
  ではなく、

**プレイヤーの行動を解釈し、
世界がどう変わったかを“物語として返す”こと。**

---

## 一番重要な考え方

### 行動そのものがプロンプトになる

プレイヤーはAIに命令文を書く必要はない。

* 何をしたか
* 何をしなかったか
* どこで迷ったか
* 何を避け続けたか

**これらの行動ログ自体が、AIへの入力（プロンプト）になる。**

AIはそれを「正しい／間違い」で評価しない。

**その人の意図・性格・立場として解釈する。**

---

## 世界は「放っておいても動く」

プレイヤーが何もしなくても、

* 時間は進み
* 状況は変化し
* 他者（または世界）は判断を下す

「不介入」「保留」「沈黙」も
**立派な選択として世界に影響する。**

---

## 同期はしないが、共有はする

* リアルタイム同期はしない
* MMOのような完全共有も目指さない

代わりに、

**他人の行動や判断を「エピソード」として受け取る。**

* 誰かが介入しなかった結果
* 別の判断をした人物の話
* 世界史の一節のような出来事

これにより、
「同じ瞬間」ではなく
**「同じ世界の痕跡」を共有する。**

---

## なぜスタンドアロンで成立するのか

この仕組みは、

* グラフィック
* 数値バランス
* 高速同期

を必要としない。

必要なのは、

* 行動ログ（テキスト）
* AIによる解釈
* 物語としての返答

つまり、
**最小構成でも“体験”が成立する。**

---

## このリポジトリの役割

現時点で ai-gm-core は：

* 実装完成を目指さない
* 正解の設計をしない
* 商用化を前提にしない

これは、
**思想を固定し、タイムスタンプを刻むための場所。**

README は世界向け。
この日本語まとめは、
**レイナとシュンさんが「何を考えてたか」を思い出すためのメモ。**
